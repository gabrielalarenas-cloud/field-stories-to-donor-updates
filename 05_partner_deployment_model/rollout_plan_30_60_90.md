# 30/60/90-Day Rollout Plan

## Goal

Design and test a responsible multilingual donor storytelling workflow for one NGO partner using ElevenLabs tools.

This rollout plan assumes a small pilot with one program country, one HQ communications team, and two to three target languages.

## First 30 days: Diagnose and design

### Objectives

- Understand the NGO's current field-to-donor storytelling workflow
- Identify one low-risk pilot use case
- Confirm consent and safeguarding requirements
- Build the first version of the workflow
- Define storyboard and voice persona before production

### Key actions

1. Map current workflow from field update to donor publication
2. Interview field, program, fundraising, and communications stakeholders
3. Identify current bottlenecks and duplication of effort
4. Select one pilot program and one recurring donor update format
5. Confirm which languages are needed
6. Review consent and safeguarding standards
7. Create a simple field update template
8. Create a human review checklist
9. Define storyboard and voice persona
10. Create first demo script using fictional or approved content
11. Generate first sample narration in ElevenLabs

### Deliverables

- Current-state workflow map
- Pilot use case definition
- Field update template
- Consent and safeguarding checklist
- Storyboard and voice persona
- First sample multilingual audio or video asset

### Success indicators

- Pilot team is aligned on scope
- Consent requirements are clear
- One sample asset is created safely
- Field team effort remains low
- Voice and tone expectations are documented

## Days 31 to 60: Pilot and measure

### Objectives

- Run a small controlled pilot
- Test the workflow with real operating conditions
- Measure speed, quality, effort, and review burden
- Identify what breaks before scaling
- Validate emotional consistency across languages

### Key actions

1. Collect 2 to 4 field updates using the agreed template
2. Review each update for consent, safety, and accuracy
3. Convert each update into a donor-ready script
4. Translate and review scripts in target languages
5. Generate narration or dubbing in ElevenLabs
6. Assemble short donor update assets
7. Track workflow time and revision cycles
8. Gather feedback from field and HQ teams
9. Compare pilot process against previous workflow
10. Document bottlenecks and improvements
11. Review whether tone, pacing, and emotional consistency are working across languages

### Deliverables

- 2 to 4 donor-ready multilingual assets
- Pilot workflow tracker
- Feedback summary from field and HQ teams
- Initial metrics dashboard
- Updated process documentation
- Voice and language quality notes

### Success indicators

- Turnaround time improves
- Field staff follow-up burden decreases
- Communications team can repeat the workflow
- Translation and review steps are manageable
- No consent or safeguarding issues arise
- Voice and tone remain credible across languages

## Days 61 to 90: Standardize and scale decision

### Objectives

- Decide whether the pilot should scale
- Convert the pilot into a repeatable operating model
- Define requirements for broader deployment
- Create partner success metrics
- Build a scale path for additional languages and country offices

### Key actions

1. Run a pilot retrospective with all stakeholders
2. Identify the highest-value use cases for scale
3. Refine consent, review, and publication workflows
4. Create templates for future donor updates
5. Define partner onboarding requirements
6. Document roles and responsibilities
7. Build a simple adoption dashboard
8. Create a training checklist for field and HQ teams
9. Estimate operational resources needed for scale
10. Recommend go / no-go / iterate decision
11. Define what would be needed to expand to additional regions or languages

### Deliverables

- Pilot retrospective
- Standard operating workflow
- Partner deployment checklist
- Adoption metrics dashboard
- Scale recommendation
- 90-day learning summary
- Regional scaling requirements

### Success indicators

- NGO can repeat the workflow without heavy support
- Stakeholders understand their responsibilities
- Success metrics are visible
- Ethical safeguards are embedded
- Scale decision is based on evidence, not excitement
- The workflow is ready for controlled expansion
