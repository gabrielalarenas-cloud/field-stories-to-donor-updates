# Partner Deployment Model for NGOs

## Purpose

This deployment model shows how an NGO could pilot an ElevenLabs-enabled field storytelling workflow in a responsible and repeatable way.

The goal is to move from an interesting AI use case to a practical operating model that nonprofit partners can actually adopt.

## Deployment principles

1. Start small
2. Use fictional or low-risk content for the first test
3. Require consent and safeguarding review
4. Keep humans in the approval loop
5. Measure operational value, not just content quality
6. Design for repeatability across countries and languages

## Target partner profile

This workflow is best suited for NGOs that:

- Operate across multiple countries
- Have field teams and HQ communications teams
- Communicate with donors in more than one language
- Need more frequent donor updates
- Have basic consent and safeguarding procedures
- Want to reduce field staff content burden
- Have a fundraising or communications team that can review final outputs

## Partner readiness criteria

| Readiness area | Question |
|---|---|
| Field operations | Do field teams already send updates, photos, or videos? |
| Communications | Does HQ regularly produce donor updates or campaigns? |
| Consent | Does the NGO have a consent process for stories and images? |
| Languages | Are updates created or received in multiple languages? |
| Review capacity | Who approves donor-facing content before publication? |
| Safeguarding | Are there rules for stories involving children or vulnerable groups? |
| Measurement | Can the team track turnaround time, effort saved, and engagement? |

## Minimum viable pilot

A small pilot should include:

- 1 program country
- 1 field coordinator
- 1 HQ communications owner
- 1 program reviewer
- 1 donor update format
- 2 to 3 languages
- 4 sample updates
- 30-day testing window

## Pilot workflow

```text
1. Partner intake
2. Use case selection
3. Field update collection
4. Consent and safeguarding review
5. Script creation
6. Translation review
7. ElevenLabs narration or dubbing
8. Final program and communications review
9. Donor publication
10. Measurement and retrospective
```

## Role responsibilities

| Role | Responsibility |
|---|---|
| Field coordinator | Submits raw update and confirms local context |
| Program lead | Reviews accuracy and program sensitivity |
| Safeguarding lead | Confirms consent and risk level |
| Communications lead | Finalizes donor-facing narrative |
| ElevenLabs / deployment owner | Supports workflow design and tool setup |
| Fundraising lead | Tracks donor engagement and usefulness |
| Operations owner | Tracks workflow time, adoption, and bottlenecks |

## Suggested partner intake questions

1. What donor communication problem are you trying to solve?
2. Which program or country would be best for a low-risk pilot?
3. What languages are involved?
4. What type of content do field teams already collect?
5. What consent process already exists?
6. Who approves donor-facing stories?
7. What content should never be used?
8. What would make this workflow useful for field staff?
9. What would make it useful for the fundraising team?
10. How will we know whether the pilot worked?

## Deployment risks

| Risk | Mitigation |
|---|---|
| Unauthorized use of someone's voice | Use only approved voices or documented consent |
| Over-polished donor narrative | Require field and program review |
| Emotional exploitation | Use dignity-first storytelling guidelines |
| Incorrect translation | Require human review in each language |
| Extra burden on field teams | Keep updates short and use existing capture habits |
| Privacy exposure | Remove names, locations, and identifying details unless approved |
| Low adoption | Start with one simple workflow and measure effort saved |

## Success definition

The pilot is successful if it:

- Reduces time from raw field update to donor-ready asset
- Reduces field staff follow-up burden
- Produces clear multilingual donor communications
- Maintains consent and safeguarding standards
- Is easy enough for the NGO to repeat
- Creates useful learning for future deployments

## Scaling path

If the pilot works, the NGO can expand by:

1. Adding more languages
2. Adding more program countries
3. Creating templates for recurring donor updates
4. Training field teams on lightweight capture guidelines
5. Creating a shared content review dashboard
6. Developing a quarterly donor storytelling calendar
7. Building a library of approved stories and voice assets
