# ElevenLabs Narration and Dubbing Workflow

## Purpose

This file describes a simple no-code workflow for using ElevenLabs to create a multilingual donor update from a reviewed field story.

This is not intended to be a production technical architecture. It is a practical proof of concept for an NGO partnership deployment.

## Tools used

- ElevenLabs Text to Speech
- ElevenLabs Dubbing or Studio
- A simple video editor such as Canva, CapCut, Loom, or Descript
- GitHub for documentation
- Optional: Google Drive or Notion for partner intake and review tracking

## Demo input

For this proof of concept, the input is a fictional field update from a fictional NGO.

No real beneficiary data, personal information, or unauthorized voice cloning is used.

## Workflow overview

```text
Raw field update
↓
Human review for consent, safety, and accuracy
↓
Script creation
↓
Translation into target languages
↓
ElevenLabs narration or dubbing
↓
Video assembly with captions
↓
Final human review
↓
Publication and measurement
```

## Step 1: Prepare the field update

The field update should be short and simple.

Example:

> This week, 42 students joined the after-school reading program. Teachers reported that several students are reading aloud with more confidence. Families also joined the first Saturday reading circle.

Before using the update, confirm that the scenario is fictional or approved for use, no identifiable beneficiary data is included, no sensitive details are included, and the update can be safely turned into a donor communication.

## Step 2: Create the donor-ready script

Transform the raw update into a short donor-facing script.

The script should be accurate, specific, clear, dignified, not exaggerated, not overly emotional, and reviewed by a human before audio generation.

## Step 3: Translate the script

Translate the script into the target languages.

For this demo:

- English
- Spanish
- French
- Optional short Swahili segment for global partnership demonstration, with native-speaker review required

Human review should confirm that the meaning is preserved across languages.

## Step 4: Generate narration in ElevenLabs

No-code option:

1. Log in to ElevenLabs
2. Open the Text to Speech tool
3. Choose a voice from the available voice library or use your own approved voice
4. Select the model appropriate to the use case
5. Paste the English script
6. Generate the audio
7. Download the audio file
8. Repeat for Spanish and French
9. Test the short multilingual demo moment if desired

Recommended file names:

```text
donor_update_en.mp3
donor_update_es.mp3
donor_update_fr.mp3
multilingual_demo_moment.mp3
```

## Step 5: Optional voice cloning

For this demo, only use your own voice or an approved ElevenLabs voice.

Do not clone the voice of a field staff member, beneficiary, child, community member, or public figure unless explicit written consent has been obtained.

For the purpose of a job application proof of work, using your own voice or a default synthetic voice is safer and more appropriate.

## Step 6: Create the video

Use Canva, CapCut, Loom, or a similar beginner-friendly editor.

Recommended structure:

```text
0:00 - Problem statement
0:15 - Raw field update
0:30 - AI-assisted multilingual workflow
0:45 - Human review and safeguarding
1:00 - Storyboard as production blueprint
1:15 - Multilingual demo moment
1:25 - Success metrics and partner scaling
```

Keep the final video between 75 and 100 seconds.

## Step 7: Add captions

Add captions in the language of the audio.

For the GitHub project, include a note that the captions are reviewed manually for accuracy.

## Step 8: Final review checklist

Before sharing the video, confirm that all content is fictional or approved, no real beneficiary data is included, no unauthorized voice cloning is used, AI-generated narration is clearly disclosed, the tone is dignified and not exploitative, the story does not exaggerate outcomes, and the process includes human review before publication.

## Output files

Recommended final assets:

```text
demo_video_link.md
donor_update_en.mp3
donor_update_es.mp3
donor_update_fr.mp3
multilingual_demo_moment.mp3
caption_script_en.srt
caption_script_es.srt
caption_script_fr.srt
```

If audio/video files are too large for GitHub, upload them to Google Drive, Loom, or YouTube as unlisted files and link them in the repository.

## What this demonstrates

This workflow shows that the project is not just a creative demo. It is a repeatable partnership deployment model that considers field staff capacity, HQ communications needs, donor engagement, multilingual accessibility, consent and safeguarding, human review, production efficiency, and scalable adoption.
