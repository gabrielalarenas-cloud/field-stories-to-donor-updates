# Success Metrics
## Measuring adoption and impact

## Purpose

This metrics framework helps an NGO evaluate whether an ElevenLabs-enabled field storytelling workflow is useful, responsible, and scalable.

The goal is not to measure whether AI can create polished content. The goal is to measure whether the workflow improves communication, reduces operational burden, and preserves consent and dignity.

## Measurement categories

### 1. Operational efficiency

| Metric | Definition | Why it matters |
|---|---|---|
| Time to donor-ready asset | Time from field update submission to approved donor asset | Measures workflow speed |
| Field staff follow-up requests | Number of clarification requests sent back to field teams | Measures field burden |
| Revision cycles | Number of review rounds before approval | Measures workflow clarity |
| Content production time | Total HQ time required to produce update | Measures internal efficiency |
| Template reuse rate | Percentage of updates using standard templates | Measures repeatability |

### 2. Partner adoption

| Metric | Definition | Why it matters |
|---|---|---|
| Active users | Number of staff using the workflow | Measures adoption |
| Update completion rate | Percentage of started updates completed | Measures usability |
| Program participation | Number of program teams submitting updates | Measures internal spread |
| Repeat use rate | Percentage of teams using the workflow more than once | Measures stickiness |
| Training completion | Percentage of users trained on workflow and safeguards | Measures readiness |

### 3. Language and accessibility

| Metric | Definition | Why it matters |
|---|---|---|
| Languages supported | Number of languages used in donor assets | Measures multilingual reach |
| Audio versions created | Number of written updates converted to audio | Measures accessibility |
| Caption completion rate | Percentage of videos with reviewed captions | Measures access and quality |
| Translation review pass rate | Percentage of translations approved without major edits | Measures quality |

### 4. Donor engagement

| Metric | Definition | Why it matters |
|---|---|---|
| Open rate | Email open rate for donor updates | Measures attention |
| Click-through rate | Percentage of donors clicking to view or listen | Measures engagement |
| Video completion rate | Percentage of viewers watching full update | Measures content relevance |
| Donor reply rate | Number of donor replies or comments | Measures connection |
| Donation conversion | Donations linked to campaign or update | Measures fundraising relevance |
| Repeat donor engagement | Engagement from existing donors over time | Measures trust |

### 5. Safeguarding and quality

| Metric | Definition | Why it matters |
|---|---|---|
| Consent completion rate | Percentage of assets with documented consent | Measures ethical readiness |
| Safeguarding review completion | Percentage of flagged stories reviewed | Measures risk control |
| Accuracy issue rate | Number of factual corrections after review | Measures reliability |
| Takedown requests | Number of requests to remove or revise content | Measures harm prevention |
| AI disclosure completion | Percentage of assets with documented AI use where required | Measures transparency |

## Example pilot dashboard

| Category | Target after 60 days |
|---|---|
| Time to donor-ready asset | 30% faster than current process |
| Field staff follow-up requests | 25% fewer clarification requests |
| Languages supported | 3 languages tested |
| Consent completion rate | 100% for all published assets |
| Revision cycles | Maximum 2 review rounds per asset |
| Partner adoption | 1 field team and 1 HQ team using workflow |
| Donor engagement | Baseline established, not optimized yet |

## What success should look like

A successful pilot does not need to prove immediate fundraising lift.

A successful pilot should prove that:

- The workflow is usable
- The content is accurate
- Consent and safeguarding controls work
- Field staff effort is reduced
- HQ teams can produce multilingual updates faster
- Donors receive clearer and more timely communication
- The process can be repeated without excessive manual coordination

## What would make the pilot fail

The pilot should not scale if:

- Field teams feel more burdened
- Consent is unclear
- Safeguarding review is skipped
- Translations are unreliable
- Stories become exaggerated or exploitative
- HQ teams cannot repeat the workflow
- AI-generated content creates trust concerns with partners or donors

## Scale decision

At the end of the pilot, choose one:

- Scale to more programs
- Continue with a second limited pilot
- Redesign the workflow
- Stop the use case due to risk or low value

The scale decision should be based on evidence, not enthusiasm for the technology.
